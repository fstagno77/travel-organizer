## Codebase Patterns
- Frontend JS usa IIFE con `window.xxx` exports — per testare in Jest, usa `new Function('window', ...)` con globals simulati
- `activityCategories.js` è caricabile in Jest Node (nessuna dipendenza DOM nel corpo principale)
- `tripPage.js` è async IIFE troppo complesso per caricare in Jest — testa la logica pura inline nel test o verifica via `fs.readFileSync` + `code.includes(...)`
- Per aggiungere un nuovo tipo di trasporto: 1) CATEGORIES in activityCategories.js, 2) getCategoryForEvent/eventToCategoryKey, 3) CATEGORY_KEYWORDS, 4) TAB_CONFIG + getVisibleTabs in tripPage.js, 5) tabRendered reset (3 posti), 6) renderTab handler, 7) container event delegation
- Il tab bar usa `segmented-label` CSS class per tabs con label lunga (trains, buses, ferries, rentals)
- `auth-reload-loop.test.js` ha un bug pre-esistente (sessionStorage non definito in Node.js) — non è rotto da noi
- `smartParser.js` usa `BETA_DOC_TYPES` per i tipi supportati in beta (incluso ferry) e `LIVE_DOC_TYPES` per prod — ferry va solo in BETA
- I nuovi tipi in smartParser: aggiungere SCHEMA, case in getSchemaForDocType, in BETA_DOC_TYPES, nel BETA_SYSTEM_PROMPT, in sanitizeResult per le date
- `getSchema(docType)` esporta lo schema come oggetto parsato (JSON.parse del SCHEMA string)
- `detectDocType(text)` è una funzione keyword-based esportata per pre-screening testo prima di Claude
